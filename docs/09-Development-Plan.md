# 09 - 开发计划与里程碑

> 本文档按阶段拆分任务，明确每个里程碑的交付物、技术重点与验收标准。

---

## 里程碑总览

| 阶段 | 周期 | 主题 | 核心交付物 |
|------|------|------|-----------|
| **M1** | 2 周 | 基础设施与认证体系 | 脚手架、CI/CD、日志监控、用户注册登录 |
| **M2** | 3 周 | 核心内容流 | 文章发布/编辑/详情、Feed 流、点赞收藏评论 |
| **M3** | 2 周 | 互动与搜索 | 关注系统、消息通知、全文搜索 |
| **M4** | 2 周 | 创作者与运营 | 创作者中心、Admin 后台、内容审核 |
| **M5** | 1 周 | 性能与稳定性 | 压测、缓存优化、降级演练、安全加固 |

---

## M1：基础设施与认证体系（第 1-2 周）

### 目标
搭建前后端项目脚手架、DevOps 流水线、可观测性基座，并完成用户认证体系。

### 任务拆分

#### 前端（Week 1-2）

| 任务 | 负责人 | 验收标准 |
|------|--------|----------|
| 初始化 Next.js 15 + TypeScript + Tailwind v4 + shadcn/ui | 前端 | 项目可运行，热更新正常 |
| 配置目录结构、路径别名、ESLint/Prettier | 前端 | 代码规范通过 CI |
| 搭建基础 UI 组件库（Button, Input, Card, Avatar, Toast） | 前端 | Storybook 或文档可预览 |
| 实现主题切换（Light/Dark） | 前端 | 切换无闪烁，Cookie 持久化 |
| 封装 HTTP Client（Ky + 拦截器） | 前端 | 统一错误拦截、Token 注入 |
| 实现登录/注册/ OAuth 页面 | 前端 | 联调通过，表单校验完善 |
| 接入 Sentry / Web Vitals（预留） | 前端 | 异常上报可用 |

#### 后端（Week 1-2）

| 任务 | 负责人 | 验收标准 |
|------|--------|----------|
| 初始化 Go Module、目录结构、Wire、Ent | 后端 | `go vet ./...` 无报错 |
| 配置 golangci-lint、单元测试基架 | 后端 | CI 中 lint 与 test 通过 |
| PostgreSQL + Redis Docker Compose 环境 | 后端 | 本地一键启动 |
| 定义 User Ent Schema 并生成代码 | 后端 | CRUD 测试通过 |
| 实现邮箱注册/登录 API | 后端 | 接口测试通过，密码 bcrypt 加密 |
| 实现 GitHub / Google OAuth 登录 | 后端 | 真实 OAuth 流程可登录 |
| JWT 签发/校验/刷新/登出 | 后端 | Token 策略符合安全规范 |
| 统一错误处理、响应封装、Recovery 中间件 | 后端 | Panic 不崩溃，返回 500 JSON |
| Zap 日志 + OpenTelemetry + Jaeger | 后端 | 链路追踪可查询 |
| Prometheus 指标埋点 + Grafana Dashboard | 后端 | 基础监控面板可用 |

#### DevOps（Week 1-2）

| 任务 | 负责人 | 验收标准 |
|------|--------|----------|
| 前端/后端 Dockerfile | DevOps | 镜像可构建 |
| GitHub Actions CI 流水线 | DevOps | PR 自动跑 lint/test/build |
| K8s Namespace + Deployment + Service YAML | DevOps | Staging 环境可部署 |
| ArgoCD 应用配置 | DevOps | 代码合并自动触发部署 |

## M2：核心内容流（第 3-5 周）

### 目标
实现文章创作与消费的核心闭环：发布、浏览、Feed 流、基础互动。

### 任务拆分

#### 前端

| 任务 | 验收标准 |
|------|----------|
| 文章发布页（Markdown + 富文本切换） | 可正常输入、预览、提交 |
| 文章详情页（SSR） | SEO Meta 正确、代码高亮、目录导航 |
| 首页 Feed 流（推荐/关注/热榜/最新） | Tab 切换流畅，骨架屏 loading |
| 点赞/收藏按钮与状态同步 |  optimistic update，数字实时变化 |
| 评论列表 + 发表评论 | 嵌套评论可展示，提交后刷新列表 |
| 用户个人主页 | 展示文章列表、基本信息 |

#### 后端

| 任务 | 验收标准 |
|------|----------|
| Post Ent Schema（文章、标签、分类） | 代码生成完毕，关系正确 |
| 文章 CRUD API | 发布、编辑、删除、查询通过测试 |
| 草稿箱逻辑 | 自动保存草稿，可恢复 |
| 内容审核状态机 | Draft → Pending → Published → Rejected |
| Feed Service 基础版 | 最新/热榜接口可用，Redis ZSet 排序 |
| 评论 CRUD API | 嵌套回复、分页可用 |
| 点赞/收藏计数 | Redis 计数 + 定时回写 PG |
| Kafka 事件流接入 | 文章发布事件可被 Search/Feed 消费 |

## M5：性能与稳定性（第 10 周）

### 目标
系统压测、性能优化、安全加固，达到上线标准。

### 任务拆分

| 任务 | 验收标准 |
|------|----------|
| 接口压测（k6 / Locust） | 核心接口 QPS > 1000，P99 < 200ms |
| 缓存策略优化 | 热点数据命中率 > 90% |
| 数据库慢查询优化 | 所有查询执行计划正常，无全表扫描 |
| 服务降级与熔断（预留） | 降级开关可用，核心功能不挂 |
| 安全扫描 | XSS、CSRF、SQL 注入防护通过渗透测试 |
| 文档补全 | API 文档、部署文档、运维手册齐全 |

---

## 每日/每周协作节奏

- **每日站会**：15 分钟同步进度与阻塞点
- **每周评审**：周五下午演示本周完成的功能
- **接口冻结**：每个 Milestone 开始前完成接口评审
- **Code Review**：所有代码合并前需至少 1 人 Review
